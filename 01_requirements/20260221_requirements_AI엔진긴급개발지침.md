# [SORI 개발팀 필독] K-Content 특화 및 비용 절감 개발 지침
> ⚠️ 기존 범용 AI 엔진 방향에서 K-Content 전문화로 즉시 전환

---

## 1. 개발 철학의 전환 — "범용성 포기, 전문성 집착"

### 핵심 타겟 장르 (3개 고정)
| 장르 | 세부 설명 |
|------|-----------|
| K-Pop (댄스/아이돌) | 화력 집중 — 인간 작곡가 수준의 디테일 |
| K-드라마 (서정적 OST) | 화력 집중 — 감성 표현 정밀도 최우선 |
| 트로트 퓨전 | 화력 집중 — 한국 정서 특화 |

### 비핵심 장르 처리 원칙
- 위 3개 외 생성 요청 → **저사양 모델로 처리 또는 후순위 배치**
- 핵심 장르에 GPU 자원 우선 배분

---

## 2. 기술적 핵심 과제 — "K-SLM(Small Language Model) 최적화"

### 모델 경량화 (Quantization & Distillation)
- **베이스 모델**: 7B 이상 금지 → **1.5B 이하 소형 모델** 사용
- **양자화**: FP16 금지 → **INT8 / INT4** 적용
- **목표 하드웨어**: RTX 3060 / 4060 등 저가형 GPU에서 상업용 음질 출력

### LoRA(Low-Rank Adaptation) 활용 전략
- 모델 전체 재학습 금지
- 장르별 **가벼운 LoRA 레이어만 교체** 방식 채택
- 기대 효과: 서버 메모리 점유율 **90% 이상 절감**
```
[K-Pop LoRA]   ─┐
[OST LoRA]     ─┤→ Base Model (1.5B, INT4)
[트로트 LoRA]  ─┘
```

---

## 3. 하드웨어 대응 전략 — "중고/리퍼비시 친화적 설계"

### 분산 배치 처리 (Batch Queue)
- 실시간 생성 대신 **'예약 생성'** 로직을 기본으로
- GPU 유휴 시간에 대량 라이브러리 생성하는 **스마트 스케줄러** 구현

### Consumer GPU 클러스터링
- 서버용 GPU 불필요
- 중고 **RTX 3090 / 4090** 여러 대를 병렬 연산 노드로 클러스터링
- 연구 우선 과제로 지정

---

## 4. B2B 상용화 패키징 — "음원이 아닌 '상품'"

### 자동 스템 분리 및 고음질화
```
생성 완료
  └→ 보컬 / 드럼 / 베이스 자동 분리 (Demucs)
       └→ AI 업스케일러 → 48kHz 24bit 리마스터링
            └→ B2B 납품 패키지 완성
```

### K-메타데이터 자동 태깅
- AI가 한국 드라마/영화 제작자 타겟 키워드 자동 추출
- 예시 키워드: `비 오는 날 카페`, `복수극 오프닝`, `새벽 독백 씬`
- 추출 결과 DB화 → 검색 엔진 연동

---

## 5. 법무적 필수 구현 사항 — "추적 가능한 AI"

### 저작권 지문 로그 (Copyright Proof)
- 모든 생성 곡 저장 시 **유사도 검사 리포트** 함께 저장 필수
- 글로벌 영화사 납품 시 법적 증빙 자료로 활용

### 데이터 가계도 (Lineage) 기록
- 기여 소스 → 혼합 비중 → 생성 결과 간 연결 로그 DB 저장
- B2B 계약 필수 증빙 자료
