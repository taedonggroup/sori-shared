# 파일 공유 규칙

> Claude Code가 정확하게 이해하려면 아래 규칙을 반드시 따라주세요.

## 1. 파일명 규칙

```
YYYYMMDD_[카테고리]_[내용].확장자

예시)
20260221_requirements_로그인기능.md
20260221_design_메인화면.png
20260221_feedback_회원가입검증.md
```

## 2. 모든 문서 파일 상단에 반드시 포함할 헤더

```markdown
---
날짜: 2026-02-21
작성자: (이름 또는 닉네임)
관련기능: (예: 로그인, 메인화면, 마이페이지)
우선순위: 높음 / 중간 / 낮음
상태: 작성중 / 완료 / 수정요청
---
```

## 3. 폴더별 파일 형식

### 01_requirements/ — 요구사항
- 형식: 마크다운(.md) 필수
- 포함 내용: 기능 설명, 사용자 시나리오, 예외 처리, 우선순위

### 02_design/ — 디자인
- 이미지 파일(png, jpg, pdf) + 반드시 동일명 설명 파일(.md) 함께 업로드
- 예) `메인화면.png` + `메인화면.md` (화면 설명 작성)

### 03_assets/ — 리소스
- 이미지, 아이콘, 폰트 파일
- `assets_index.md`에 파일 목록과 용도 기록

### 04_feedback/ — 피드백
- 검증 결과는 REVIEWER.md 형식 준수
- 버그는 GitHub Issues로 직접 등록

### 05_reference/ — 참고자료
- 형식 자유, 출처 명시 필수

## 4. SUMMARY.md 업데이트 (필수)

파일 올릴 때마다 `SUMMARY.md`에 한 줄 추가:

```markdown
| 2026-02-21 | 로그인 요구사항 | 01_requirements/ | 완료 |
```
